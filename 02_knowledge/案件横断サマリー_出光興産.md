## 見積パターン分析

見積書ナレッジ（144ファイル）の分析から、以下の主要パターンが抽出された。

### パターン1: クラウドアーキ基盤移行（最多パターン）

館林DC廃止に伴う、オンプレミス→Azure移行の見積パターン。

**典型的なフェーズ構成:**
1. 調査・要件定義
2. 設計
3. 製造
4. 結合テスト
5. システムテスト
6. ユーザテスト
7. 本番稼働
8. 管理

**工数・単価の傾向:**
- 担当者単価: 100万円/人月
- 管理者単価: 140万円/人月
- 1システムあたり: 約4〜21人月（システム規模による）

**代表的な見積例（工場系6システム一括）:**

| システム | 工数（人月） | 金額 |
|---|---|---|
| 数量確定システム（千葉・徳山） | 20.7 | 約2,222万円 |
| 用役管理システム（千葉・徳山） | 13.6 | 約1,464万円 |
| 千葉化学品質管理システム | 14.8 | 約1,588万円 |
| LMPP包装システム | 8.3 | 約906万円 |
| 工場DB | 0.9 | 約110万円 |
| ヤード操業システム（千葉・徳山） | 5.7 | 約606万円 |
| **合計** | **64.0** | **約6,896万円** |

**前提条件（共通）:**
- 業務・機能変更は行わない（移行のみ）
- サーバ構成は現行踏襲
- DBは現行踏襲（Oracle、SQL Server継続）
- JavaフレームワークはStruts継続
- インフラ環境構築は出光様にて実施
- データ移行は出光様にて実施

**移行時の主要技術作業:**
- AP基盤: Oracle WebLogic → Azure App Service (PaaS)
- ear → warファイルへのビルド手順変更（build.xml修正）
- EJBの対応（ラッパー化等）
- 認証基盤: Oracle WebLogic → Azure Entra ID
- IP/ホスト名変更に伴う向け先変更

### パターン2: 縮小共通基盤移行

クラウドアーキ基盤移行の前段として、館林DCから縮小版の共通基盤への移行パターン。

**特徴:**
- OS/MWのバージョン変更を伴う
- 現行共通基盤の構成を踏襲
- クラウドアーキ基盤移行への橋渡し的位置づけ

**適用案件:** 事務管理システム、次世代研SWAN

### パターン3: 新規開発・機能追加

IFC生産委託ポータル、MES千葉工場等の開発案件。

**典型的なフェーズ構成:**
- 基本設計 → 詳細設計 → 製造単体 → 結合試験 → IT → 1W → 2W

**規模区分:** A（大）/ D（中）/ X（対象外）等のランク付けで工数を算出

**JCC/Exe分担パターン:**
- セルイン作成、セルアウト作成、販売計画処理、成長確認、DashBoards等の機能単位で分担
- Exe様担当分とJCC担当分を明確に分離

### パターン4: ESG/SSBJ対応（構想策定型）

ESGシステム導入構想書に基づく提案型パターン。

**特徴:**
- Wave1（環境分先行、2026年7月稼働目標）→ Wave2（SSBJフル版、2027年4月稼働）の段階アプローチ
- パッケージソリューション（Envizi等）の標準機能を最大限活用
- RFPによる製品評価を実施
- Step1〜Step3の段階的導入アプローチ

### パターン5: XenApp廃止・Webアプリ化

XenApp（Citrix）廃止に伴い、既存の.NET/VBアプリをWebアプリケーションへ刷新するパターン。

**適用案件:** MA-ADS Webアプリ化、XenApp廃止アプリ移行
